# Design: backend-plugins

## Context

`MacBackend` (SPI-trait: `probe`, `get_user_mnkc`, `apply_session`, файловые
метки + fd-вариант) остаётся открытым policy-контрактом. Ранее ParsecBackend
подключался compile-time feature `astra-mac`; после этой миграции открытый
host всегда один, а закрытый libpdp FFI поставляется runtime-плагином.
Жёсткое ограничение: `pdp_set_pid` работает только с pid=0 (подтверждено
по pdp.h на живой Astra) — установка метки обязана происходить В процессе
PAM-сессии, поэтому плагин загружается in-process (`dlopen`), а вариант
«бэкенд-демон» невозможен для МКЦ.

Плагины — канал поставки коммерческих бэкендов (см. spec `licensing-distribution`);
публичная плагин-экосистема — non-goal. Мульти-ключи и стабильность ABI поддерживаются
best-effort и не являются блокерами развития SPI.

## Goals / Non-Goals

**Goals:**
- Один открытый бинарь дистрибуции; платные бэкенды — подписанные `.so`.
- Kind-типизированный конверт: v1 = `backend.enforcement`; механизм не требует редизайна
  для будущих видов.
- Верификация подписи до dlopen, политика в бинарнике (не в конфиге).
- Чистая миграция с `--features astra-mac` без изменения SPI-trait'а.

**Non-Goals:**
- Публичный SDK/обещание стабильности ABI наружу.
- Kinds `login.method` (методы входа остаются compile-time крейтами ядра) и `ui.companion`
  (greeter-артефакты — плагины к чужим хостам: их грузит fly-dm/gnome-shell; для них механизм
  даёт только упаковку/доставку) — зарезервированы, не реализуются в v1.
- Hot-reload/unload плагинов.

## Decisions

1. **C-ABI vtable, не abi_stable, не IPC.** `#[repr(C)]`-структуры, entry
   `extern "C" fn tessera_plugin_entry() -> *const TesseraPluginHeader`.
   Альтернативы: `abi_stable` (тяжёлая зависимость, Rust-only плагины, сложнее аудит);
   процесс-плагин с IPC (невозможно: apply_session на вызывающий процесс).
   Прецедент долгоживущего C-ABI плагин-интерфейса — kgreeterplugin (2003, не менялся).
2. **Заголовок**: `{abi_version: u32, kind: u32, name, plugin_version: *const c_char,
   vtable: *const c_void}`. `abi_version` — строгое равенство (предохранитель; семантика —
   «стараемся не ломать», bump при необходимости с lockstep-поставкой плагинов).
   `kind`: 1 = backend.enforcement (vtable = C-зеркало MacBackend +
   init(config)/teardown, МРД-probe и generic MAC-write capability preflight).
3. **Политика доверия в бинарнике.** Конфигурируемая политика отвергнута:
   конфиг лежит на том же диске с теми же правами — атакующий, способный
   подложить `.so`, правит и конфиг. Release: список raw Ed25519 public keys
   (32 байта, 64 hex chars, через запятую) вшивается при сборке
   (`TESSERA_PLUGIN_PUBKEYS` — ключ вендора + опционально заказчика);
   detached-файл имеет формат `ed25519:<128 hex>` и подписывает точные байты
   `.so`. Верификация идёт до `dlopen`, отключения нет. Official CI не
   допускает release-сборку с пустым списком. Debug не верифицирует
   (`cfg(debug_assertions)`); `.deb` всегда release. Open-source допускает
   сборку со своим списком ключей.
4. **Без автоактивации.** Каталог `/usr/lib/tessera/plugins/` (root:root 0755/0644)
   сканируется, но грузится только бэкенд, явно названный конфигом (`[mac] backend =
   "parsec"`); не назван → StubBackend; лишние файлы — audit-событие, не загрузка.
5. **Жизненный цикл**: verify → dlopen(`RTLD_NOW|RTLD_LOCAL`) → entry → abi/kind-проверка →
   `init(config-блоб)` → регистрация; живёт до конца процесса (dlclose в PAM-процессе
   не делается). Каждый callback ставит `catch_unwind` внутри самого
   плагина и возвращает `PLUGIN_PANIC`; unwind между независимо связанными
   `cdylib` runtimes запрещён. Host отображает статус в ошибку PAM-вызова
   и событие `plugin_panic`.
6. **Мост**: `PluginBackend` реализует trait `MacBackend` поверх C-vtable — orchestrator,
   политика и тесты ядра не меняются. Фикстурный тест-плагин собирается в CI.
7. **Конфиг init v1**: ABI принимает opaque byte slice, но Parsec v1 не
   требует backend-specific настроек, поэтому host передаёт пустой blob.
   Формат будущего непустого blob вводится отдельной версией контракта,
   а не неявным парсингом всей секции `[mac]`.

## Модель угроз

Канон — tessera-ws/specs/threat-model.md §5.6. Сводка: плагин = исполняемый код в
root-процессе логина — самая чувствительная точка; защита от A3/A4 (подлог .so на диске,
канал доставки) — вшитый список ключей + подпись над байтами; конфиг-флип невозможен
(политика в бинарнике); A6/полный доступ к диску (подмена самого бинаря) — вне модели
(целостность среды, подписанные пакеты).

## Risks / Trade-offs

- [ABI-дрейф между бинарём и плагином] → строгое равенство abi_version: несовпадение =
  отказ загрузки + audit, никогда «авось совместимо»; lockstep-релизы.
- [Официальный бинарь не грузит самосборные плагины заказчика] → ключ заказчика в списке
  при enterprise-сборке либо пересборка из исходников; не engineering-блокер.
- [dlopen в PAM-процессе: символьные конфликты с хост-приложением (sshd, fly-dm)] →
  `RTLD_LOCAL`, минимум зависимостей плагина (статическая линковка кроме libc/libpdp),
  e2e на реальных PAM-хостах.
- [Паника/крэш плагина валит логин] → panic ловится внутри plugin callback
  и возвращается числовым статусом; аппаратный fault/abort доверенного
  нативного кода изолировать in-process невозможно и он остаётся отказом входа.
- [Удлинение пути логина (verify+dlopen)] → демон загружает выбранный backend
  один раз при старте; PAM загружает его при открытии сессии. Негативные
  результаты не кэшируются между процессами.

## Migration Plan

1. `plugin-loading` в ядре + фикстурный тест-плагин (открытая часть, без поведенческих изменений).
2. tessera-enterprise: `tessera_backend_parsec` (cdylib) — обёртка существующего ParsecBackend;
   подпись в пайплайне; enterprise .deb = плагин + .sig (+ зависимость от открытого пакета).
3. Конфиг `[mac] backend = "parsec"` в Astra-поставке; e2e на VM.
4. Удаление feature `astra-mac`; CI-матрица: один бинарь + сборка плагинов.
После (4) откат выполняется возвратом всего change, параллельной статической
схемы в целевой архитектуре нет.

## Resolved Questions

- Подпись v1 — Ed25519; algorithm id обязателен в detached-файле, чтобы
  следующий ABI мог добавить ГОСТ без двусмысленного разбора старых подписей.
- Config blob v1 — opaque bytes; Parsec получает пустой blob.
