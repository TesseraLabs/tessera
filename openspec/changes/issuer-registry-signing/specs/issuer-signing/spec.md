## ADDED Requirements

### Requirement: Подпись реестра выделенным ключом
Агент (`issuer serve`) ДОЛЖЕН (MUST) поддерживать подпись объединённого
реестра устройств отдельной операцией с выделенным ключом реестра
(`--registry-key`), отличным от ключа выпуска. Ключом реестра агент
НЕ ДОЛЖЕН (MUST NOT) подписывать ничего, кроме реестров; ключом выпуска —
НЕ ДОЛЖЕН (MUST NOT) подписывать реестры. При совпадении меток ключа выпуска
и ключа реестра агент ДОЛЖЕН (MUST) отказаться стартовать. Без настроенного
ключа реестра операция подписи реестра ДОЛЖНА (MUST) отвечать явной ошибкой
«ключ не настроен». Подпись — ECDSA P-256 поверх SHA-256 от байтов payload,
совместимая с существующей проверкой снапшотов в кабинете.

#### Scenario: Подпись реестра при настроенном ключе
- **WHEN** агент запущен с `--registry-key`, отличным от ключа выпуска, и кабинет запрашивает подпись объединённого реестра
- **THEN** агент возвращает подпись P-256 ключом реестра, и кабинет собирает файл, проходящий существующую проверку подписи

#### Scenario: Ключ реестра совпадает с ключом выпуска
- **WHEN** агент запускается с `--registry-key`, равным метке ключа выпуска
- **THEN** агент отказывается стартовать с внятной ошибкой

#### Scenario: Ключ реестра не настроен
- **WHEN** кабинет запрашивает подпись реестра у агента без `--registry-key`
- **THEN** агент отвечает ошибкой «ключ не настроен», кабинет предлагает экспорт без подписи
