# issuer-cabinet Specification

## Purpose

Веб-кабинет выпуска: один статический SPA (WASM-сборка ядра `tessera_issuer`)
без серверной логики. Вся сборка TBS и проверки — на клиенте, стейт — файлы,
работает self-hosted и офлайн; доступные операции выводятся из предъявленного
родительского сертификата. Родство с CLI `issuer` — общее ядро и одни и те же
проверки. Этот раздел фиксирует поведение кабинета: serverless-инварианты,
режим-из-родителя, инвентарь для форм, источник ключа листа, локализацию и
паритет с CLI.

## MODIFIED Requirements

### Requirement: Инвентарь для форм — снапшот файлом с проверкой подписи

Источником данных форм (устройства, теги, роли) ДОЛЖЕН (MUST) служить инвентарь,
полученный одним из двух способов: загрузкой снапшот-файла ЛИБО сборкой в самом
кабинете (конструктор). Собранный в кабинете инвентарь эквивалентен
неподписанному (ручному) снапшоту. Если загруженный снапшот подписан, подпись
ДОЛЖНА (MUST) проверяться; снапшот с невалидной подписью ДОЛЖЕН (MUST)
отвергаться. Кабинет ДОЛЖЕН (MUST) отображать происхождение инвентаря
(подписанный экспорт | заполнен вручную) и его возраст. Отсутствие инвентаря
НЕ ДОЛЖНО (MUST NOT) блокировать выпуск — поля заполняются вручную с явной
пометкой.

При наличии инвентаря формы выпуска ДОЛЖНЫ (MUST) использовать его как источник
подсказок, не ограничивая оператора: устройства (`host_binding`) и пользователи
(`user_binding`) предлагаются из инвентаря, но оператор МОЖЕТ (MAY) ввести
значение, которого в инвентаре нет. Если инвентарь перечисляет роли, набор ролей
в форме листа ДОЛЖЕН (MUST) быть пересечением рамок родителя (`allow_roles`
envelope) с ролями инвентаря, и сужение относительно рамок родителя ДОЛЖНО (MUST)
быть заметно оператору; пустой список ролей в инвентаре роли НЕ ограничивает
(эквивалент «инвентарь роли не задаёт») — предлагаются полные рамки родителя.
Инвентарь НЕ ДОЛЖЕН (MUST NOT) расширять выбор за пределы рамок родителя —
envelope остаётся потолком, инвентарь только сужает и подсказывает.

#### Scenario: Битая подпись снапшота
- **WHEN** загружен снапшот, заявленный как подписанный, с невалидной подписью
- **THEN** снапшот отвергается, формы не заполняются его данными

#### Scenario: Ручной снапшот
- **WHEN** загружен неподписанный снапшот (ручной)
- **THEN** формы заполняются, источник помечен как ручной, возраст показан

#### Scenario: Сборка инвентаря конструктором
- **WHEN** оператор собирает инвентарь в кабинете (устройства, пользователи, роли, теги) без загрузки файла
- **THEN** инвентарь принимается как ручной (неподписанный), помечен как заполненный вручную, и питает подсказки форм наравне с загруженным снапшотом

#### Scenario: Инвентарь питает пикеры форм, envelope — потолок
- **WHEN** предъявлен CA организации и загружен (или собран) инвентарь, роли которого — подмножество рамок CA
- **THEN** форма листа предлагает устройства и пользователей из инвентаря (с возможностью ввести своё), а список ролей равен пересечению рамок CA с ролями инвентаря; роль вне рамок CA не предлагается, даже если присутствует в инвентаре
