# mrd-guard — контракт «МРД не поддерживается» + защита от даунгрейда метки

## Why

Tessera назначает сессии только МКЦ (целостность). Поведение на системах с
активным МРД (мандатное разграничение доступа по конфиденциальности, Astra
уровень защищённости «Смоленск») до сих пор не было ни специфицировано, ни
тестировано: parsec-метка — единый объект с обеими осями (libpdp записывает
её только целиком), а контракт «запись метки не затрагивает ось
конфиденциальности» нигде не был зафиксирован. Нужен явный контракт
«МРД-системы не поддерживаются в v0.x» и два защитных механизма вместо
полной поддержки.

## What Changes

- **Контракт**: системы с активным МРД официально не поддерживаются; поведение
  фиксируется спекой, а не остаётся неопределённым.
- **startup-check**: новая проверка `mac_mrd_active` — детект активного МРД на
  устройстве. `[mac].runtime=required` → ERROR (демон не стартует);
  `auto` → WARN; `disabled` → INFO. Способ детекта подтверждается на Astra VM
  (кандидаты: libpdp-запрос максимального уровня конфиденциальности,
  `astra-mac-control status`).
- **Сохранение поля конфиденциальности**: применение метки сессии НЕ ДОЛЖНО
  (MUST NOT) изменять поле 0 (конфиденциальность) существующей метки процесса —
  read-modify-write вместо записи нуля. Правка в коммерческом crate
  `tessera_mac_parsec` (кодек/FFI), контракт — в открытом SPI `mac-integrity`.
- **Доки**: абзац в `docs/mac-integrity.md` о границе МКЦ/МРД и статусе
  «МРД не поддерживается».

## Capabilities

### New Capabilities

Нет — изменение укладывается в существующие capabilities.

### Modified Capabilities

- `mac-integrity`: новый контракт SPI — применение метки сессии сохраняет поле
  конфиденциальности (MUST NOT изменять поле 0); явное требование «МРД вне
  области ответственности Tessera» (назначается штатными механизмами ОС).
- `daemon-lifecycle`: startup-check pipeline дополняется проверкой
  `mac_mrd_active` с политикой ERROR/WARN/INFO по `[mac].runtime`.

## Impact

- Открытый репозиторий: `tessera_cli/src/startup_check/` (новая проверка +
  расширение probe-SPI), `tessera_core/src/mac/` (сигнатура/доки SPI, если
  контракт требует), `docs/mac-integrity.md`.
- Коммерческий backend `tessera_mac_parsec` (отдельный трек внедрения):
  МРД-probe, кодек меток и запись меток по контракту сохранения поля
  конфиденциальности.
- `cli-diagnostics` (`tessera check`) получает новую проверку автоматически —
  требование описывает pipeline целиком, отдельная delta не нужна.
- Тесты: unit на кодек (roundtrip с ненулевым полем 0), unit на политику
  ERROR/WARN/INFO, e2e-подтверждение на Astra VM — отдельным шагом.
