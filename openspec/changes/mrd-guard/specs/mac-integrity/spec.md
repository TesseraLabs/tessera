# mac-integrity — delta (mrd-guard)

## ADDED Requirements

### Requirement: МРД вне области ответственности

Tessera НЕ ДОЛЖНА (MUST NOT) назначать, выбирать или изменять уровень
конфиденциальности (МРД, поле 0 parsec-метки) сессий, файлов или дескрипторов.
SPI и label-алгебра остаются осью целостности (МКЦ). Системы с активным МРД
не поддерживаются; их обнаружение — обязанность startup-check
(см. [daemon-lifecycle](../daemon-lifecycle/spec.md)).

#### Scenario: Система с активным МРД

- **WHEN** Tessera работает на устройстве с активным МРД
- **THEN** Tessera не участвует в назначении конфиденциальности; гарантируется
  только отсутствие вмешательства в поле 0 и сигнал startup-check

### Requirement: Сохранение поля конфиденциальности при записи меток

Любая запись parsec-метки backend'ом (применение метки сессии, установка метки
файла или дескриптора) НЕ ДОЛЖНА (MUST NOT) изменять поле конфиденциальности
(поле 0) существующей метки цели: перед записью текущая метка ДОЛЖНА (MUST)
быть прочитана, и её поле 0 перенесено в записываемую метку без изменений.
Запись константного нуля в поле 0 запрещена.

#### Scenario: Применение mac_mask на МРД-системе

- **WHEN** штатный логин присвоил сессии ненулевой уровень конфиденциальности,
  и Tessera применяет `mac_mask` роли к сессии
- **THEN** ось целостности метки установлена по `mac_mask`, поле
  конфиденциальности осталось равным назначенному логином значению

#### Scenario: Обычная система без МРД

- **WHEN** метка процесса имеет поле конфиденциальности 0 (МРД неактивен)
- **THEN** запись метки даёт прежний результат — поведение на поддерживаемых
  конфигурациях не меняется
