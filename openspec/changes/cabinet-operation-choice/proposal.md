# Proposal: cabinet-operation-choice

## Why

Кабинет жёстко привязывал операцию выпуска к типу родителя (корень → только CA
организаций, CA организации → только листы), хотя модель ядра позволяет любому
CA с рамками делегирования выпускать и подчинённые CA, и листы — wasm-ядро
кабинета уже поддерживает обе операции от любого родителя с рамками.
UI-ограничение не давало пользоваться этим и расходилось с ядром.

## What Changes

- Для родителя-CA с рамками делегирования (корень или CA организации) кабинет
  предлагает обе операции выпуска — переключателем; по умолчанию сохраняется
  прежнее поведение (корень → CA организации, CA организации → лист смены).
- Секция CRL по-прежнему доступна любому CA-родителю; лист/непригодный
  сертификат — операции недоступны с объяснением (без изменений).
- Формат сертификатов, рамки делегирования и wasm-ядро не меняются —
  правка только UI кабинета (SPA) и документации.

Breaking-изменений нет.

## Capabilities

### New Capabilities

Нет.

### Modified Capabilities

- `issuer-cabinet`: требование «Один кабинет — режим из родительского
  сертификата» — доступные операции по-прежнему выводятся из предъявленного
  родителя, но CA с рамками даёт выбор из двух операций выпуска, а не одну
  жёстко по типу.

## Impact

- `cabinet/src/ui/app.ts` (секция операции + состояние выбора),
  новый `cabinet/src/ui/operations.ts` + тесты, `cabinet/src/i18n/dict.ts`,
  `cabinet/public/styles.css`.
- `docs/{ru,en}/issuer.md` — три места про вывод операций из родителя.
- Wasm-ядро, протокол агента, спеки issuer-signing — не затронуты.
