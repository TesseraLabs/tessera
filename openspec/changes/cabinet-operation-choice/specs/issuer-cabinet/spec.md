# Delta: issuer-cabinet

## MODIFIED Requirements

### Requirement: Один кабинет — режим из родительского сертификата

Кабинет ДОЛЖЕН (MUST) быть единственным (один SPA, одна сборка) и выводить
доступные операции из предъявленного родительского сертификата. Родитель-CA с
рамками делегирования (корень парка или CA организации) даёт обе операции
выпуска — подчинённого CA организации с назначением суженных рамок и листа
смены строго внутри рамок родителя; операция выбирается переключателем.
По умолчанию ДОЛЖНА (MUST) предлагаться операция, соответствующая типу
родителя: корень парка → выпуск CA организации, CA организации → выпуск листа
смены. Отдельных сборок или режимов «по ролям пользователей» НЕ ДОЛЖНО
(MUST NOT) существовать; формы и валидация — общие.

#### Scenario: Предъявлен корень парка
- **WHEN** оператор загружает корневой сертификат парка
- **THEN** кабинет предлагает переключатель операций с выбранным по умолчанию выпуском CA организаций с назначением рамок; выпуск листа доступен переключением

#### Scenario: Предъявлен CA организации
- **WHEN** оператор загружает CA организации
- **THEN** кабинет предлагает переключатель операций с выбранным по умолчанию выпуском листов смен; выпуск подчинённого CA доступен переключением; формы ограничены рамками этого CA (значения вне рамок не предлагаются и отвергаются)

#### Scenario: Смена родителя сбрасывает выбор операции
- **WHEN** оператор с выбранной вручную операцией загружает другой родительский сертификат
- **THEN** выбор операции сбрасывается на умолчание для типа нового родителя

#### Scenario: Предъявлен лист или непригодный сертификат
- **WHEN** загруженный сертификат не является CA (или не содержит рамок делегирования)
- **THEN** операции выпуска недоступны, кабинет объясняет причину
