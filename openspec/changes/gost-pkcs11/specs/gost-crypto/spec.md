# gost-crypto Delta Specification

## ADDED Requirements

### Requirement: Хост-верификация GOST-подписи токена

Подпись, выполненная токеном на PKCS#11-пути GOST-механизмом, ДОЛЖНА (MUST) верифицироваться
на хосте через gost-engine (Streebog-256/512 по NID публичного ключа из серта: 979/980) —
симметрично PKCS#12-пути; собственная криптография по-прежнему запрещена (SHALL NOT).
Загрузка engine — ленивая (`ensure_loaded_if_any_gost`), недоступность engine при GOST-ключе →
отказ (fail-closed). Raw-подпись токена (`r||s`, 64/128 байт) ДОЛЖНА (MUST) перекодироваться
в форму, ожидаемую gost-engine, до верификации; перекодировка покрывается тестами на
эталонных векторах. Если выбран механизм «raw-подпись по дайджесту», Streebog-дайджест
ДОЛЖЕН (MUST) считаться на хосте тем же gost-engine (дайджест не секрет; приватный ключ
не покидает токен).

#### Scenario: Верификация подписи токена
- **WHEN** токен вернул GOST-подпись nonce (2012-256 или 2012-512)
- **THEN** подпись перекодируется и верифицируется gost-engine'ом публичным ключом из серта; провал → `BadSignature` (fail-closed)

#### Scenario: Engine недоступен при GOST-токене
- **WHEN** на токене GOST-ключ, gost-engine не загружается
- **THEN** `EngineLoadFailed` → отказ верификации; self_check ловит это раньше при `needs_gost()`
